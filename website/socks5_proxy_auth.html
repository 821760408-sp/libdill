NAME

socks5_proxy_auth - Authenticate an incoming client proxy connection request

SYNOPSIS

#include <libdill.h>

int socks5_proxy_auth(
    int s,
    int (*)(const char *username, const char *password) auth_function,
    int64_t deadline);

DESCRIPTION

WARNING: This is experimental functionality and the API may change in the future.

SOCKS5 is an application-level protocol described in RFC 1928. SOCKS5 protocol is used to negotiate a connection via a proxy server. The underlying transport must be a bytestream.

This function listens for an incoming client authentication request, negotiates an authentication method and authenticates the client.

s: Handle of the underlying socket. It must be a bytestream protocol.

auth_function: authorization callback function to validate usernate and password. If auth_function is NULL, clients may connect without authentication.

deadline: A point in time when the operation should time out, in milliseconds. Use the now function to get your current point in time. 0 means immediate timeout, i.e., perform the operation if possible or return without blocking if not. -1 means no deadline, i.e., the call will block forever if the operation cannot be performed.

No socket is created.

This function is not available if libdill is compiled with --disable-sockets option.

RETURN VALUE

In case of success the function returns 0. In case of error it returns -1 and sets errno to one of the values below.

ERRORS

  • EACCES: The client is not authorized to connect.
  • EBADF: Invalid handle.
  • ECANCELED: Current coroutine was canceled.
  • ECONNRESET: Broken connection.
  • ENOTSUP: The handle does not support this operation.
  • EPROTO: Error in the client <-> proxy protocol.
  • ETIMEDOUT: Deadline was reached.

EXAMPLE

int s = tcp_connect(&proxy_addr, -1);
int err = socks5_client_connect(s, &uname, &passwd, &remote_addr, -1);
if(err) return -1;
bsend(s, "ABC", 3, -1);
char buf[3];
brecv(s, buf, sizeof(buf), -1);
tcp_close(s);
